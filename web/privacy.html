Privacy, in plain language.

Last updated July 21, 2026

Public-beta draft: the operator must add a real contact method and review this policy before inviting the public. This page describes the software's current behavior; it is not a legal-compliance claim.

What Clip Vault processes

Clip Vault receives Discord account and server identity details needed to sign you in and determine which servers you may access. It stores a session identifier in a secure, HTTP-only cookie. Discord OAuth access and refresh tokens are not retained.

Voice and clips

While a bot admin has recording active, the bot holds a rolling in-memory voice buffer. The server chooses either a visible notice with member opt-out or explicit opt-in. Members can use the notice buttons or /privacy allow and /privacy block; these choices apply to future clips, and blocking also clears that member's current buffered audio. Only audio selected by /clipthat is encoded and saved.

On a saved clip, a speaker can remove their own voice from every playable revision. Clip Vault immediately blocks stale audio URLs, renders replacement mixes, and replaces the bot's current Discord attachment. The per-speaker source remains private so that person can later create a separate personal cut; doing so never changes the shared original or reverses another person's removal. /privacy remove-past provides a confirmed, batched removal across existing clips. Copies that other people downloaded before removal cannot be recalled.

Stored data

Saved clips can include audio, titles, speaker and creator identifiers, edit revisions, favorites, activity records, server settings, delegated admins, and member recording preferences. Server names and icons are synchronized from Discord. Operational logs do not intentionally include tokens or audio contents.

Retention and deletion

Server admins choose active-clip retention. Favorited clips are protected from automatic retention, but authorized users may trash them. Trash and its revisions are permanently removed after 30 days. The Discord server owner can export metadata or permanently erase the server's Clip Vault data from Manage.

Who can see data

Access is limited to authenticated members of the relevant Discord server. Clip creators and bot admins have additional edit actions; bot admins can see revision history and trash. Every API action checks permissions independently of the visible interface.

Security and hosting

Production sessions use Secure, HTTP-only, SameSite cookies, CSRF protection, origin checks, security headers, rate limits, and server-side authorization. No internet service can promise perfect security. A production operator is responsible for encrypted backups, updates, incident response, and publishing a support contact.
